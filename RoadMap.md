# Nota 编译器开发路线图

本项目旨在将 Nota 语言（一种类似 QML 的前端 UI 语言）编译成可在 Web 浏览器中渲染的 HTML 和 CSS 代码。开发遵循工业级标准，确保最终产品具有专业水准的稳定性、可扩展性和性能。

## Phase 1: 项目基础与核心分析 (已完成)

- [x] **项目初始化**
  - [x] 创建 `RoadMap.md` 来规划和记录开发进程。
  - [x] 搭建基于 CMake 的 C++20 项目结构。
  - [x] 创建 `src`, `tests`, `scripts` 等标准化目录。
  - [x] 集成 Google Test 测试框架。

- [x] **词法分析器 (Lexer)**
  - [x] 实现词法分析器，将 Nota 源代码分解为 Token 流。
  - [x] 支持的 Token：标识符、大括号、冒号、字符串、数字、百分比、注释等。
  - [x] 编写全面的单元测试，覆盖所有 Token 类型和边界情况。

- [x] **抽象语法树 (AST) 定义**
  - [x] 定义用于表示 Nota 组件层级结构、属性和关系的 C++ 数据结构。
  - [x] 设计节点类型，如 `ComponentNode`, `PropertyNode`, `ValueNode` 等。

- [x] **语法分析器 (Parser)**
  - [x] 实现一个递归下降解析器，用于将 Token 流转换为 AST。
  - [x] 设计了强大的错误报告机制。
  - [x] 编写了全面的单元测试，验证 AST 结构的正确性。

## Phase 2: 代码生成与核心功能 (已完成)

- [x] **语义分析**
  - [x] 实现符号表，用于跟踪组件和属性的定义。
  - [x] 验证属性的合法性（例如，`spacing` 仅在 `Row` 和 `Col` 中有效）。
  - [x] 实现了基础的类型检查。

- [x] **HTML/CSS 代码生成器**
  - [x] 设计了一个代码生成器，遍历 AST 并输出等效的 HTML 和 CSS 代码。
  - [x] 实现对基本组件 (`App`, `Row`, `Col`, `Rect`, `Text`) 的转换。
  - [x] 将 Nota 属性映射为对应的 CSS 样式。

- [x] **命令行界面 (CLI)**
  - [x] 创建了一个命令行工具，接收 `.nota` 文件作为输入，并生成 `.html` 文件。
  - [x] 支持 `--output` 参数。

## Phase 3: 高级语言特性 (已完成)

- [x] **自定义组件 (`Item`)**
  - [x] 支持使用 `Item` 关键字定义新的可复用组件。
  - [x] 实现组件实例化和属性传递。

- [x] **属性覆写与子组件访问**
  - [x] 支持在实例化组件时覆写其默认属性。
  - [x] 实现对 `this.children[index].property` 形式的子组件属性访问和修改。

- [x] **相对定位与布局**
  - [x] 实现对 `x`, `y` 属性的支持，生成 `position: absolute` 和 `left`, `top` 样式。
  - [x] 确保父组件自动应用 `position: relative` 样式。
  - [ ] **(待办)** 完整实现 `position` 属性，支持 `left top`, `center` 等定位方式。

- [x] **层级控制 (`index`)**
  - [x] 实现 `index` 属性，并将其转换为 CSS 的 `z-index` 属性。

## Phase 4: 生产环境准备 (已完成)

- [x] **错误处理与诊断**
  - [x] 完善了编译过程中的错误报告，提供清晰、准确的错误信息（包括行号和列号）。

- [x] **性能优化**
