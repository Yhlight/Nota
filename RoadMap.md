# Nota 编译器开发路线图

本项目旨在将Nota语言（一种类似QML的前端UI语言）编译成可在Web浏览器中渲染的HTML和CSS代码。开发将遵循工业级标准，确保最终产品具有专业水-准的稳定性、可扩展性和性能。

## Phase 1: 项目基础与核心分析 (预计完成时间: YYYY-MM-DD)

- [x] **项目初始化**
  - [x] 创建 `RoadMap.md` 来规划和记录开发进程。
  - [x] 搭建基于CMake的C++20项目结构。
  - [x] 创建 `src`, `tests`, `scripts` 等标准化目录。
  - [x] 集成Google Test测试框架。

- [x] **词法分析器 (Lexer)**
  - [x] 实现词法分析器，将Nota源代码分解为Token流。
  - [x] 支持的Token：标识符、大括号、冒号、字符串、数字、百分比、注释等。
  - [x] 编写全面的单元测试，覆盖所有Token类型和边界情况。

- [x] **抽象语法树 (AST) 定义**
  - [x] 定义用于表示Nota组件层级结构、属性和关系的C++数据结构。
  - [x] 设计节点类型，如 `ComponentNode`, `PropertyNode`, `ValueNode` 等。

- [x] **语法分析器 (Parser)**
  - [x] 实现一个递归下降或Pratt解析器，用于将Token流转换为AST。
  - [x] 初步实现能够解析基本的组件结构（如 `App { ... }`）。
  - [x] 设计错误报告机制，为后续的语义分析和编译阶段提供支持。
  - [x] 编写单元测试，验证AST结构的正确性。

## Phase 2: 代码生成与核心功能 (预计完成时间: YYYY-MM-DD)

- [x] **语义分析**
  - [x] 实现符号表，用于跟踪组件和属性的定义。
  - [x] 验证属性的合法性（例如，`spacing` 仅在 `Row` 和 `Col` 中有效）。
  - [x] 类型检查，确保属性值的类型正确（如 `width` 应为数字或百分比）。

- [x] **HTML/CSS 代码生成器**
  - [x] 设计一个代码生成器，遍历AST并输出等效的HTML和CSS代码。
  - [x] 实现对基本组件 (`App`, `Row`, `Col`, `Rect`, `Text`) 的转换。
  - [x] 将Nota属性（如 `width`, `color`, `padding`）映射为对应的CSS样式。
  - [x] 将组件的层级关系转换为HTML的DOM树结构。

- [x] **命令行界面 (CLI)**
  - [x] 创建一个简单的命令行工具，接收 `.nota` 文件作为输入，并生成 `.html` 文件。
  - [x] 支持 `--output` 参数，允许用户指定输出文件的路径。
