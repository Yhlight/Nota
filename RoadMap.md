# Nota 编译器开发路线图

本项目旨在创建一个工业级水准的 Nota 编译器，将 `.nota` 文件编译为高性能的 HTML 和 CSS 代码。

## 阶段一：项目奠基与 MVP (最小可行产品)

*   **目标**: 搭建项目基础架构，完成最核心的编译功能。
*   **时间线**: 2-4 周

| 任务 | 描述 | 状态 |
| --- | --- | --- |
| **环境搭建** | 配置 C++20 编译环境, 设置 CMake, 集成 GoogleTest 测试框架。 | 待办 |
| **项目结构** | 创建 `src` 和 `tests` 目录, 规划模块结构。 | 待办 |
| **词法分析器** | 实现 Lexer, 将 Nota 源代码分解为 Token 流。 | 待办 |
| **语法分析器** | 实现 Parser, 将 Token 流构建为抽象语法树 (AST)。 | 待办 |
| **代码生成器** | 实现 Generator, 将 AST 转换为 HTML 和 CSS 字符串。 | 待办 |
| **基础组件** | 支持 `App`, `Row`, `Col`, `Rect`, `Text` 组件的编译。 | 待办 |
| **基础属性** | 支持 `width`, `height`, `color` 等核心 CSS 属性。 | 待办 |
| **快照测试** | 建立第一个快照测试, 验证端到端编译流程。 | 待办 |

## 阶段二：高级语言特性

*   **目标**: 实现 Nota 语言的进阶功能，增强其表达能力。
*   **时间线**: 4-6 周

| 任务 | 描述 | 状态 |
| --- | --- | --- |
| **自定义组件** | 实现 `Item` 组件的定义与使用。 | 待办 |
| **属性覆写** | 支持 `this.property = value` 的语法。 | 待办 |
| **子控件访问** | 支持 `this.children[index]` 的语法。 | 待办 |
| **相对定位** | 实现 `x`, `y` 和 `position` 属性。 | 待办 |
| **层级控制** | 实现 `index` 属性 (z-index)。 | 待办 |
| **更多属性** | 扩展支持 `spacing`, `padding`, `border`, `radius` 等。 | 待办 |

## 阶段三：健壮性与优化

*   **目标**: 提升编译器的错误处理能力、性能和代码质量。
*   **时间线**: 3-5 周

| 任务 | 描述 | 状态 |
| --- | --- | --- |
| **错误处理** | 提供清晰、准确的编译错误和警告信息。 | 待办 |
| **性能分析** | 对编译器进行性能分析, 识别并优化瓶颈。 | 待办 |
| **代码重构** | 遵循 C++ Core Guidelines, 提升代码可读性和可维护性。 | 待办 |
| **单元测试** | 提高 Lexer, Parser, Generator 各模块的单元测试覆盖率。 | 待办 |

## 阶段四：工具链与生态

*   **目标**: 提供完善的开发者工具，构建 Nota 生态。
*   **时间线**: 长期

| 任务 | 描述 | 状态 |
| --- | --- | --- |
| **命令行工具** | 开发功能丰富的 CLI, 支持文件监视、批量编译等。 | 待办 |
| **官方文档** | 撰写详细的 Nota 语言规范和编译器使用文档。 | 待办 |
| **打包与发布** | 提供跨平台的预编译版本。 | 待办 |
| **社区建设** | 构建开发者社区, 收集反馈, 持续迭代。 | 待办 |
