# Nota 编译器开发路线图

本文档规划了 Nota 编译器的开发过程，旨在交付一个专业、稳定且可维护的编译器，能够将 Nota 语言编译为生产环境级别的 HTML 和 CSS 代码。

## 开发阶段

我们将开发过程分为以下几个主要阶段：

### Phase 1: 项目基础架构与核心模块 (Project Foundation & Core Modules)

- **目标**: 搭建稳固的项目结构，定义核心数据模型。
- **任务**:
    1.  **项目设置**:
        - [ ] 初始化 C++ 项目 (使用 CMake)。
        - [ ] 建立清晰的目录结构 (`src`, `include`, `tests`, `scripts`)。
    2.  **核心数据结构定义**:
        - [ ] 定义词法单元 (Token) 结构。
        - [ ] 设计抽象语法树 (AST) 节点，用于表示组件、属性和层级关系。
    3.  **错误处理模块**:
        - [ ] 设计一个集中的错误报告系统，用于在编译的各个阶段收集和显示错误信息。
    4.  **构建与测试集成**:
        - [ ] 配置构建系统 (CMake)。
        - [ ] 集成一个单元测试框架 (如 GoogleTest)。

### Phase 2: 词法分析器 (Lexer)

- **目标**: 将 Nota 源代码文本转换为词法单元 (Token) 序列。
- **任务**:
    1.  **Token 识别**:
        - [ ] 识别关键字 (e.g., `App`, `Row`, `Item`)。
        - [ ] 识别标识符 (e.g., `id`, `width`)。
        - [ ] 识别字面量 (数字, 字符串, 颜色值)。
        - [ ] 识别符号 (e.g., `{`, `}`, `:`, `;`)。
        - [ ] 处理注释 (`//`, `/*...*/`)。
    2.  **单元测试**:
        - [ ] 编写全面的单元测试，确保所有语法元素都能被正确解析。

### Phase 3: 语法分析器 (Parser)

- **目标**: 将 Token 序列转换为抽象语法树 (AST)。
- **任务**:
    1.  **AST 构建**:
        - [ ] 实现一个递归下降或 Pratt 解析器，用于处理组件的嵌套结构。
        - [ ] 解析组件定义 (`App { ... }`)。
        - [ ] 解析属性键值对 (`width: 100%`)。
    2.  **语法错误处理**:
        - [ ] 实现稳健的错误恢复机制，以便在遇到语法错误时能继续解析并报告多个错误。
    3.  **单元测试**:
        - [ ] 针对有效的和无效的 Nota 代码片段编写测试，验证 AST 的结构是否正确，以及错误是否被正确捕获。

### Phase 4: 语义分析与符号表 (Semantic Analysis & Symbol Table)

- **目标**: 验证 AST 的语义正确性，并收集符号信息。
- **任务**:
    1.  **类型检查**:
        - [ ] 验证属性值的类型是否正确 (e.g., `width` 应该是数字或百分比)。
    2.  **符号解析**:
        - [ ] 构建符号表，管理组件 ID。
        - [ ] 解析 `parent` 和 `this` 关键字的引用。
    3.  **自定义组件处理**:
        - [ ] 实现对 `Item` 定义的自定义组件的解析。
        - [ ] 处理组件实例化和属性覆写。
    4.  **语义错误报告**:
        - [ ] 报告未定义的属性、类型不匹配、ID 冲突等错误。

### Phase 5: 代码生成器 (Code Generator)

- **目标**: 将经过验证的 AST 转换为最终的 HTML 和 CSS 代码。
- **任务**:
    1.  **HTML 生成**:
        - [ ] 遍历 AST，为每个 Nota 组件生成对应的 HTML 元素 (`<div>`, `<span>`, `<body>`)。
        - [ ] 分配 CSS 类名和 ID。
    2.  **CSS 生成**:
        - [ ] 从 AST 节点的属性中提取样式信息。
        - [ ] 将 Nota 属性 (e.g., `width`, `color`, `spacing`) 转换为标准 CSS 规则。
        - [ ] 处理 `x`, `y`, `position` 和 `index` 属性，生成对应的 `position`, `left`, `top`, `z-index` 等 CSS。
        - [ ] 优化 CSS，例如通过合并重复的类来减少代码冗余。
    3.  **文件输出**:
        - [ ] 将生成的 HTML 和 CSS 代码写入到 `.html` 和 `.css` 文件中。

### Phase 6: 命令行工具 (CLI) 与工具链

- **目标**: 提供一个用户友好的命令行界面来调用编译器。
- **任务**:
    1.  **CLI 开发**:
        - [ ] 创建一个主执行文件。
        - [ ] 实现命令行参数解析 (输入文件、输出目录等)。
    2.  **文件 I/O**:
        - [ ] 实现对 `.nota` 文件的读取和输出文件的写入。
    3.  **集成**:
        - [ ] 将词法分析、语法分析、语义分析和代码生成等所有模块集成到 CLI 工具中。

### Phase 7: 高级功能与优化

- **目标**: 实现 Nota 语言的全部功能并进行性能优化。
- **任务**:
    1.  **表达式求值**:
        - [ ] 实现对属性表达式的计算 (e.g., `x: parent.width / 2`)。
    2.  **子控件访问**:
        - [ ] 实现对 `this.children[index]` 的访问和修改。
    3.  **性能优化**:
        - [ ] 优化编译速度。
        - [ ] 优化生成的 CSS 和 HTML 的性能。
    4.  **文档**:
        - [ ] 编写详细的编译器使用文档和语言规范。
