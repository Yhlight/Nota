# Nota 编译器开发路线图

本文档旨在规划和记录 Nota 编译器的开发过程，确保最终交付一个专业、稳定且可用于工业生产环境的高质量编译器。整个开发过程将分为以下几个核心阶段：

## 第一阶段：项目奠基 (Project Setup)

*   **目标：** 为编译器项目搭建一个坚实、可扩展的基础架构。
*   **任务：**
    1.  **环境初始化：** 创建 C++20 项目结构。
    2.  **构建系统配置：** 使用 CMake 管理项目构建、依赖和编译选项。
    3.  **测试框架集成：** 集成 GoogleTest 作为单元测试和功能测试框架。
    4.  **代码规范与工具：** 引入 Clang-Format 和 Clang-Tidy，确保代码风格和质量的一致性。
    5.  **基础命令行界面 (CLI)：** 实现一个简单的 CLI，能够接收输入文件路径并打印提示信息。

## 第二阶段：词法分析 (Lexical Analysis)

*   **目标：** 将输入的 Nota 源代码字符串分解为一系列有意义的词法单元 (Tokens)。
*   **任务：**
    1.  **定义词法单元：** 明确 Nota 语言的所有 Token 类型（如标识符、大括号、冒号、字符串、数字等）。
    2.  **实现词法分析器 (Lexer)：** 开发一个能够读取 Nota 源码并输出 Token 流的 Lexer。
    3.  **编写单元测试：** 针对 Lexer 编写完备的测试用例，覆盖所有 Token 类型及各种边界情况。

## 第三阶段：语法分析 (Syntactic Analysis)

*   **目标：** 基于词法单元流，构建能够准确反映代码逻辑结构的抽象语法树 (AST)。
*   **任务：**
    1.  **设计 AST 结构：** 定义 AST 节点的具体形式，如组件节点、属性节点等。
    2.  **实现语法分析器 (Parser)：** 开发 Parser，消费 Token 流并构建 AST。
    3.  **实现错误处理：** 为语法错误（如括号不匹配、缺少属性值等）提供清晰、准确的错误提示。
    4.  **编写单元测试：** 验证 Parser 能否为合法的 Nota 代码片段构建正确的 AST，并能正确处理非法输入。

## 第四阶段：代码生成 (Code Generation)

*   **目标：** 遍历抽象语法树，生成最终的 HTML 和 CSS 代码。
*   **任务：**
    1.  **实现代码生成器 (Generator)：** 开发一个 Generator，负责遍历 AST。
    2.  **组件到 HTML 的映射：** 将 Nota 的内置组件 (`App`, `Row`, `Col` 等) 转换为对应的 HTML 标签和 CSS 类。
    3.  **属性到 CSS 的转换：** 将 Nota 属性 (`width`, `color`, `spacing` 等) 翻译为具体的 CSS 规则。
    4.  **建立快照测试 (Snapshot Testing)：** 编译一系列 `.nota` 示例文件，将其输出与预先定义好的、正确的 HTML/CSS 快照进行比对，以确保代码生成的准确性。

## 第五阶段：高级特性实现 (Advanced Features)

*   **目标：** 实现 `Nota.md` 中定义的自定义组件、属性覆写、相对定位等高级功能。
*   **任务：**
    1.  **自定义组件 (`Item`)：** 支持 `Item` 的定义和实例化。
    2.  **属性覆写与子控件访问：** 实现 `this.property` 和 `this.children[index].property` 的解析与赋值逻辑。
    3.  **相对定位：** 处理 `x`, `y` 和 `position` 属性，确保父组件自动添加 `position: relative`。
    4.  **层级控制：** 实现 `index` 属性到 `z-index` 的转换。

## 第六阶段：工具链与优化 (Toolchain & Optimization)

*   **目标：** 完善编译器的配套工具，提升易用性，并对输出产物进行优化。
*   **任务：**
    1.  **强化 CLI：** 增加对输出目录、编译选项等的支持。
    2.  **产物优化：** (可选) 实现 CSS 的合并与压缩，减小最终产物的体积。
    3.  **性能分析：** 对编译器本身进行性能剖析和优化。
    4.  **文档完善：** 撰写详细的开发与使用文档。
